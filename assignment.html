Job Interview Assignment

Python SQL Docker PostgreSQL FastAPI

This assignment involves the development of a data engineering solution that integrates weather forecast data from the Meteomatics Weather API into PostgreSQL and exposes the data through FastAPI endpoints that run complex dynamic SQL queries to retrieve the requested aggregated data. The process includes the use of Docker for containerization ensuring consistent environments and facilitating easy scaling, secure handling of sensitive information using environment variables, and sample tests that validate the functionality of each endpoint.

Task
- Create a program to retrieve 7-day weather forecasts for any 3 locations using the Meteomatics API.
- Store the data in a relational database with an appropriate schema.
- Develop an API that leverages the database to provide endpoints to list the following:
    • The latest forecasts for each location daily
    • The average of the last 3 forecasts for each location daily
    • The top n locations based on each forecast metric

Implementation
This project utilizes Docker Compose to set up services for the Meteomatics Weather API Client, PostgreSQL, and FastAPI, enabling seamless integration across various operating systems and facilitating straightforward deployment on diverse devices. The Meteomatics Weather API Client manages data ingestion for three locations based on user input, implementing best practices for error handling through sophisticated try-and-except blocks and ensuring sensitive information security with environment variables. Subsequently, a dynamic Python script transforms the data into the appropriate schema by parsing the JSON response from the API and loading it into PostgreSQL tables.

Furthermore, FastAPI was chosen to create the various endpoints outlined in the task description. In the backend, there is a combination of Python and multiple complex dynamic SQL queries, capable of extracting data from PostgreSQL tables and executing the necessary functionality for each user-specified city. Each endpoint response must follow a predefined structure and the calculations are validated through tests that are based on static data, ensuring accuracy and data quality.
